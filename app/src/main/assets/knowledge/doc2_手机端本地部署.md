# 手机端本地部署

手机端本地部署指的是将大模型与语音能力直接运行在用户手机上，而不依赖云端服务器。随着移动芯片算力提升与量化技术成熟，这种做法在隐私、时延与离线可用性上具备明显优势。

本地部署的第一步是选择合适的模型与量化格式。以 GGUF 为例，它支持多种量化级别（如 Q4_K_M、Q5_K_M），能在几乎不损失效果的前提下把模型体积压缩到手机可承载的范围。配合 llama.cpp 等推理框架，可在仅使用 CPU 的情况下完成流畅对话。

部署流程通常包括：拉取推理引擎源码（如以 git submodule 方式引入）、将模型文件放入应用私有目录、在首启动时把模型从安装包资源拷贝到本地存储，并记录版本号以便增量更新。模型文件不应打入 APK 主体，而应通过资源目录或用户自行下载放置，以减少包体。

性能调优方面，建议将推理线程数限制为 CPU 核心数与 4 的较小值，避免抢占系统资源导致发热与卡顿。上下文窗口可根据内存情况在 4096 到 8192 之间权衡。对于内存紧张的设备，应优先选择更低比特的量化。

本地部署还意味着权限与安全的重新设计：录音仅在端侧用于识别，模型权重不上传，整个链路默认离线。只有在用户主动开启“联网兜底”时，才允许在特定情况下访问网络，从而把隐私控制权交还给用户。
