# Phi-3 模型介绍

Phi-3 是微软推出的一系列小型语言模型（SLM），主打“小参数、高性能”。其中 Phi-3-mini 仅约 38 亿参数，却在大量常识推理与语言理解基准上接近体量数倍于它的模型，非常适合在手机、笔记本等端侧设备部署。

Phi-3-mini 提供 4K 与 128K 两种上下文版本。移动端常用 4K 指令版（Phi-3-mini-4K-Instruct），配合 ChatML 对话模板：用户消息以 `<|user|>` 包裹，助手消息以 `<|assistant|>` 包裹，段末以 `<|end|>` 标记。模型在训练阶段已充分学习该格式，因此按模板拼装即可获得稳定对话效果。

为了能在资源受限的设备上运行，Phi-3 通常以 GGUF 格式提供量化权重，例如 Q4_K_M。量化在几乎不损失质量的前提下显著减小体积与内存占用，使模型可在纯 CPU 推理框架（如 llama.cpp）中流畅加载。

在端侧推理时，Phaser 的参数选择会影响体验：温度（temperature）控制随机性，top_p 控制候选截断范围，二者配合可在“稳定”与“发散”之间取得平衡。上下文窗口建议设为 4096，内存充足时可上探至 8192。

Phi-3 的优势在于“够用且省资源”，非常适合作为离线 AI 对话、文档问答、语音助手背后的大脑。结合本地知识库与轻量 ASR，即可构建一条完全不依赖网络的智能对话流水线。
